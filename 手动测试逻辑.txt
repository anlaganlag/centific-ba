手动测试步骤

  前置条件

  1. 配置 API Keys — 编辑 backend/.env，填入真实的：
    - AZURE_OPENAI_API_KEY 和 AZURE_OPENAI_ENDPOINT（AI 分析用）
    - OPENAI_API_KEY（文档嵌入用，标准 OpenAI，非 Azure）
  2. Docling Serve — .env 中已配置了远程实例 URL，用于 PDF/DOCX 解析。如果只测 .txt 文件则不需要。

  启动服务

  打开两个终端：

  终端 1 — 后端（端口 8001）：
  cd backend
  uvicorn app.main:app --reload --port 8001

  终端 2 — 前端（端口 5173）：
  cd frontend
  npm run dev

  然后浏览器打开 http://localhost:5173

  测试流程

  └──────┴───────────────────────────────────────────┴──────────────────────────────────────────────────────────┘

  注意事项

  - 前端 Vite 配置会将 /api 代理到 http://localhost:8001，所以前后端端口不同没问题
  - 上传 PDF/DOCX 需要远程 Docling Serve 可达；上传 .txt 不需要
  - 完整的 Analysis pipeline 需要 Azure OpenAI 和 OpenAI 两个 key 都配好
  - 数据存储在 backend/data/ 下（SQLite + ChromaDB + uploads），已被 .gitignore 排除
